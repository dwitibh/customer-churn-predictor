How this works

Model & pipeline

Logistic Regression trained on four numeric features from the customer churn dataset: Age, Gender, Tenure, MonthlyCharges. Targets binary Churn (Yes / No). Gender is encoded as Female = 1, Male = 0. Inputs are standardised with StandardScaler before fitting.

scikit-learnStandardScalerLogisticRegressionL2, C=1.080/20 split

Model comparison (notebook)

Five algorithms were compared with GridSearchCV (cv=5). Logistic Regression and SVC tied on accuracy. LR was chosen for this demo because it ports to client-side JS cleanly and gives interpretable per-feature contributions.

Live scoring in the browser

The trained logistic regression weights and the scaler's mean / std are hardcoded as JSON. Each prediction runs as:

z = w₀ + Σ wᵢ · (xᵢ − μᵢ) / σᵢ
p = σ(z) = 1 / (1 + e−z)

Driver contributions are wᵢ · (xᵢ − μᵢ) / σᵢ for each feature. Positive contributions push the probability up, negative pull it down. No backend, no Python runtime — everything runs in the user's browser.

Caveats

This is a 1,000-row public dataset with a churn rate near 80%, so a "predict churn" baseline already lands around 70-80% accuracy. What this project demonstrates is the pipeline, not the absolute numbers: data cleaning, encoding, scaling, model comparison with cross-validation, hold-out evaluation, and porting a trained model to a live browser-side scorer with feature-level explanations.

⚠ Public-data caveat. 1,000-row customer churn dataset. The notebook trains five models (LR, KNN, SVC, Decision Tree, Random Forest) on four features. This page reproduces the logistic regression pipeline and runs it client-side. Insights are illustrative of the pipeline rather than commercial conclusions.
